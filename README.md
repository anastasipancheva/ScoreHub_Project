# ScoreHub: Система оптимизации учебного процесса

**ScoreHub** — это веб-платформа для автоматизации проведения занятий, управления очередями на защитах и прозрачного расчета академического рейтинга. Спроектировано специально под реалии курса «Дискретные структуры».

## О проекте

Проект решает проблему «хаоса» на парах с перевернутым классом и контрольных точках:
- **Автоматизация очередей:** Студенты не ждут в коридоре, а видят свой статус и вызов ассистента в реальном времени.
- **Гибкий расчет баллов:** Система учитывает групповые коэффициенты (0.8–1.2), штрафы за дедлайны и сложные мультипликаторы КТ.
- **Mobile-First:** Интерфейс оптимизирован для работы со смартфонов прямо во время аудиторных занятий.

## Технологический стек

* **Frontend:** Angular (Signals, RxJS, SCSS)
* **Backend:** .NET 8 (C#, Entity Framework Core, MediatR)
* **Real-time:** SignalR (для мгновенных уведомлений и живой очереди)
* **Database:** PostgreSQL
* **Design:** Custom Clean UI

## 📋 Основные роли

1.  **Студент:**
    * Прохождение входных мини-тестов (первые 5 минут пары).
    * Просмотр списка команд и закрепленных ассистентов.
    * Кнопка «Позвать ассистента» для консультации или сдачи.
    * Мониторинг личного прогресса.
2.  **Ассистент:**
    * Управление списком команд/задач.
    * Прием задач с выставлением коэффициентов за работу пары.
    * Вызов студентов из очереди по хронологии или приоритету задачи.
3.  **Преподаватель (Админ):**
    * Конфигурация параметров предмета через JSON-структуры.
    * Настройка весов задач, мультипликаторов и порогов оценок.
    * Назначение ассистентов на конкретные группы.
    * Прием задач с выставлением коэффициентов за работу пары.
    * Вызов студентов из очереди по хронологии или приоритету задачи.

## Архитектура данных

Система использует нормализованную структуру БД для поддержки гибких правил:
- **LessonParticipation:** хранит бонусный коэффициент (0.8–1.2), применяемый ко всей лекции.
- **QueueItem:** обеспечивает хронологический порядок сдачи (кто первый нажал — того первого вызвали).
- **AssistantAssignment:** позволяет динамически привязывать ассистентов к задачам на КТ или к командам на лекциях.

## Математика баллов

Итоговый балл модуля рассчитывается по формуле:
`Module_Score = (Sum(TaskPoints * GroupCoeff + MiniTest) + Sum(HW * TimeCoeff)) * KT_Multiplier`
* **GroupCoeff:** 0.8 — 1.2 (оценка работы команды ассистентом).
* **TimeCoeff:** 0.75 (до 7 дней), 0.5 (после 7 дней).
* **KT_Multiplier:** мультипликатор от 0.5 до 2.0 на основе решенных задач на КТ.
